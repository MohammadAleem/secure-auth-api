# Contributing to Secure Auth API

Thank you for your interest in contributing!

---

## 🚀 Getting Started

1. Fork the repository
2. Create a new branch using: git checkout -b feature/your-feature-name
3. Make your changes
4. Commit using proper format:
   - feat: describe what you added
   - fix: describe what you fixed
   - docs: describe what you documented
5. Push your branch and open a Pull Request

---

## 📋 Rules

- Never commit directly to `main`
- One feature or fix per Pull Request
- Always write a clear PR description
- Test your changes before submitting

---

## 🐛 Reporting Bugs

Open an Issue with:
- What you expected to happen
- What actually happened
- Steps to reproduce
- Your OS and Python version

---

## 💡 Suggesting Features

Open an Issue with the label `enhancement` and describe:
- The problem it solves
- How you imagine it working
